Enfoque Prevencion y remediacion

Las recomendaciones se centran en hardening, validacion, monitoreo y mantenimiento seguro.

A01

Broken Access Control

Alto

Fallas que permiten a usuarios o procesos actuar fuera de los permisos previstos, acceder a datos no autorizados o modificar recursos que no les corresponden.

Ejemplos

  • Paneles administrativos expuestos sin controles adecuados.
  • Archivos o rutas sensibles visibles desde internet.
  • Permisos inconsistentes entre frontend, API y backend.

Senales VigiScan

  • Directorios expuestos como admin, backup o rutas de configuracion.
  • Headers ausentes que reducen aislamiento de navegador.
  • Tecnologias conocidas con CVE asociadas a control de acceso.
Relacion: rutas expuestas, headers, tecnologias y CVE.
Recomendacion: aplicar autorizacion del lado servidor, minimo privilegio, revisiones de rutas sensibles y pruebas de acceso por rol.
A02

Security Misconfiguration

Alto

Configuraciones inseguras, incompletas o por defecto que dejan servicios, frameworks o servidores web con una superficie innecesariamente amplia.

Ejemplos

  • Headers de seguridad ausentes o debiles.
  • Listados de directorio, archivos temporales o backups visibles.
  • Mensajes de error demasiado detallados.

Senales VigiScan

  • Content-Security-Policy, HSTS o X-Frame-Options faltantes.
  • Rutas comunes sensibles expuestas.
  • Versiones de servidor o framework reveladas.
Relacion: headers, tecnologias detectadas y rutas expuestas.
Recomendacion: usar baselines de hardening, desactivar defaults inseguros, limitar informacion expuesta y revisar configuracion por ambiente.
A03

Software Supply Chain Failures

Alto

Riesgos introducidos por dependencias, paquetes, imagenes, librerias o componentes externos que no estan inventariados, verificados o mantenidos.

Ejemplos

  • Frameworks desactualizados publicados en produccion.
  • Componentes sin seguimiento de versiones.
  • Uso de librerias con vulnerabilidades conocidas.

Senales VigiScan

  • Tecnologias identificadas por cabeceras, cookies o HTML.
  • Coincidencias con CVE locales asociadas a producto y version.
  • Archivos de paquetes o configuracion visibles accidentalmente.
Relacion: tecnologias, CVE y rutas expuestas.
Recomendacion: mantener SBOM, actualizar dependencias, verificar origen de artefactos y monitorear CVE relevantes.
A04

Cryptographic Failures

Alto

Proteccion insuficiente de datos sensibles por cifrado debil, transporte inseguro, secretos expuestos o politicas incompletas de seguridad del navegador.

Ejemplos

  • Ausencia de HTTPS obligatorio o HSTS.
  • Secretos o archivos de entorno accesibles.
  • Cookies sin atributos de proteccion adecuados.

Senales VigiScan

  • Strict-Transport-Security ausente.
  • Rutas como .env o configuraciones publicas.
  • Tecnologias con CVE criptograficas conocidas.
Relacion: headers, rutas expuestas, tecnologias y CVE.
Recomendacion: forzar HTTPS, proteger secretos, revisar cookies, rotar claves y usar algoritmos y librerias vigentes.
A05

Injection

Alto

Datos no confiables llegan a interpretes, consultas o comandos sin validacion y separacion adecuadas. Esta guia evita payloads y se limita a clasificacion defensiva.

Ejemplos

  • Entradas no validadas en formularios o parametros.
  • Errores que revelan detalles internos de consultas o servicios.
  • Componentes con CVE relacionadas con validacion de entrada.

Senales VigiScan

  • CVE conocidas para tecnologia y version detectadas.
  • Headers faltantes que agravan inyeccion de contenido en navegador.
  • Mensajes de error o rutas que sugieren exposicion de detalles internos.
Relacion: tecnologias, CVE, headers y rutas expuestas.
Recomendacion: validar entradas, usar APIs parametrizadas, codificar salidas, revisar errores y priorizar parches de componentes afectados.
A06

Insecure Design

Medio

Debilidades originadas en decisiones de arquitectura, flujos o controles ausentes desde el diseno, incluso si la implementacion no contiene un bug puntual.

Ejemplos

  • Ausencia de limites claros para acciones sensibles.
  • Flujos sin separacion de roles o aprobaciones.
  • Controles de seguridad agregados tarde o de forma parcial.

Senales VigiScan

  • Patrones repetidos de headers ausentes en varios objetivos.
  • Superficie publica amplia con rutas sensibles recurrentes.
  • Tecnologias heredadas que condicionan el modelo de seguridad.
Relacion: headers, tecnologias y rutas expuestas.
Recomendacion: modelar amenazas, definir controles por flujo, documentar decisiones y validar requisitos defensivos antes de produccion.
A07

Authentication Failures

Alto

Fallas en identificacion de usuarios, manejo de sesiones o proteccion de credenciales que facilitan acceso indebido o persistencia no autorizada.

Ejemplos

  • Paneles de login o administracion expuestos sin controles adicionales.
  • Sesiones con configuracion debil de cookies.
  • Versiones de framework con problemas conocidos de autenticacion.

Senales VigiScan

  • Rutas login, admin o paneles encontrados en superficie publica.
  • Headers defensivos ausentes en paginas sensibles.
  • CVE asociadas a tecnologias de autenticacion o sesion.
Relacion: rutas expuestas, headers, tecnologias y CVE.
Recomendacion: aplicar MFA donde corresponda, endurecer cookies, limitar accesos administrativos y monitorear eventos de autenticacion.
A08

Software or Data Integrity Failures

Medio

Fallas para asegurar que codigo, datos, actualizaciones o artefactos no fueron alterados de forma no autorizada antes de ser usados por la aplicacion.

Ejemplos

  • Actualizaciones sin verificacion de integridad.
  • Artefactos generados por CI/CD sin controles de confianza.
  • Backups o archivos de datos disponibles publicamente.

Senales VigiScan

  • Backups, archivos comprimidos o datos expuestos.
  • Tecnologias con historial de CVE de integridad.
  • Headers que no reducen riesgo de manipulacion de contenido.
Relacion: rutas expuestas, tecnologias, CVE y headers.
Recomendacion: firmar artefactos, verificar integridad, proteger backups y separar permisos de despliegue.
A09

Logging & Alerting Failures

Medio

Eventos de seguridad no se registran, no se alertan o no se conservan con suficiente contexto para detectar, investigar y responder a incidentes.

Ejemplos

  • No registrar accesos fallidos o cambios administrativos.
  • Alertas inexistentes para errores repetidos o cambios criticos.
  • Logs con datos sensibles visibles para usuarios no autorizados.

Senales VigiScan

  • VigiScan no confirma logging interno, pero ayuda a priorizar donde revisar.
  • Rutas sensibles expuestas que deberian tener monitoreo reforzado.
  • CVE y tecnologias criticas que requieren alertas operativas.
Relacion: rutas expuestas, tecnologias y CVE.
Recomendacion: centralizar logs, definir alertas de seguridad, proteger datos registrados y revisar cobertura de eventos criticos.
A10

Mishandling of Exceptional Conditions

Medio

Manejo insuficiente de estados inesperados, errores, fallos de dependencias o condiciones anormales que pueden revelar informacion o dejar el sistema en estado inseguro.

Ejemplos

  • Errores tecnicos visibles al usuario final.
  • Fallas que no cierran de forma segura.
  • Respuestas inconsistentes ante recursos inexistentes o incompletos.

Senales VigiScan

  • Errores devueltos durante el escaneo de rutas comunes.
  • Versiones reveladas por respuestas o paginas de error.
  • Headers ausentes que aumentan el impacto de respuestas inesperadas.
Relacion: rutas expuestas, tecnologias y headers.
Recomendacion: usar manejo centralizado de errores, fallar de forma segura, registrar excepciones y mostrar mensajes genericos al usuario.
